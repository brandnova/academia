# Code of Conduct

## Our commitment

Academia exists to help students find and preserve useful knowledge. The way we work together on this project should reflect that same spirit: helpful, patient, and focused on making the thing better.

This applies to every space connected to the project: the GitHub repository (issues, pull requests, discussions, commit messages), the project chat, and any other channel used for Academia-related collaboration.

## Expected behavior

- Be respectful and constructive in feedback. Critique the code or the idea, not the person.
- Be patient with contributors who are newer to Git, GitHub, Django, or Next.js. Several people on this project are learning these tools for the first time, that's expected and welcome, not a shortcoming.
- Ask questions in the open (in the issue or the chat) rather than assuming, so answers help the next person too.
- Give credit where it's due, and accept credit gracefully when it's given to you.
- Assume good intent, and when a disagreement happens, focus on resolving it, not winning it.

## Unacceptable behavior

- Personal attacks, insults, or demeaning comments directed at another contributor.
- Dismissing or mocking someone for their experience level.
- Harassment of any kind, including unwelcome contact outside the agreed project channels.
- Deliberately submitting misleading, plagiarized, or malicious code.
- Publicly disclosing another contributor's private information without their consent.

## Scope

This Code of Conduct applies to all project spaces, and also applies when someone is officially representing the project in public spaces (for example, posting about Academia on social media in a project-affiliated capacity).

## Reporting

If you experience or witness behavior that violates this Code of Conduct, report it directly to the project maintainer (Nova) via the project chat or a direct message. Reports will be handled privately and taken seriously.

## Enforcement

The project maintainer is responsible for clarifying and enforcing this Code of Conduct, and may take any action deemed appropriate, including a warning, temporary removal from project spaces, or permanent removal from the project, in response to behavior that violates it.

## Attribution

This Code of Conduct was written for Academia, drawing on common practices from the wider open source community, adapted to fit a small team that includes first-time contributors.
